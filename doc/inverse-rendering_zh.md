# 基于预计算标准事件的逆向渲染

本文档描述了一种拟议的替代渲染方法，用确定性的逐像素计算取代当前的蒙特卡洛 (MC) 管线，
基于预计算的标准事件 (standard events)。该思路受 Gislén 等人 (2004) 的旋转协变性理论启发，
但适配于平行光（太阳）冰晕模拟场景。

**状态**: 理论设计阶段，尚未实现。

## 动机

当前 MC 管线以"正向"方式工作：采样晶体朝向 → 光线追踪 → 记录出射光命中的像素 → 累积。
这本质上是有噪声的；收敛需要大量采样，且计算是"浪费性"的——光线被分散到所有像素上，
无论是否需要。

提议的方法将此反转：对每个像素，确定哪些晶体朝向对其有贡献，并解析地计算贡献量。
这完全消除了 MC 噪声，并将昂贵的光线追踪预计算（每种晶体形状做一次）与廉价的渲染步骤
（每组参数做一次）分离。

### 与 Beam Tracing 的关系

Beam tracing 实验（`exp/beam_tracing` 分支，参见 [beam-tracing.md](beam-tracing.md)）
尝试消除**入射点采样**方差。其核心发现是：入射点方差可忽略不计——真正的瓶颈是
SO(3) 上的**朝向采样**。

逆向渲染方法直接解决这个瓶颈：不再采样朝向并期望收敛，而是对每个像素精确计算所有
有贡献的朝向集合。

## 理论基础

### 旋转协变性

晶体散射分布满足旋转协变性 (Gislén eq. 2)：

```
p(b̂ | â, U) = p(U⁻¹b̂ | U⁻¹â, 1)
```

其中 `â` 是入射方向，`b̂` 是散射方向，`U` 是晶体朝向（SO(3) 旋转），`1` 是标准
（单位）朝向。

含义：任何朝向为 `U` 的晶体产生的散射事件，都可以通过将一个**标准事件**（晶体在
单位朝向下）旋转 `U` 得到。

### 标准事件

标准事件是一对 `(â₀, b̂₀)`，其中：
- `â₀` 是标准朝向晶体上的一个随机入射方向
- `b̂₀` 是光线追踪得到的散射方向
- 散射角为 `ω = arccos(â₀ · b̂₀)`
- 关联权重 `w` 包含 Fresnel 系数和投影面积

对于给定的晶体形状，标准事件可以密集预计算并存储。

### 逐像素反演

对于输出图像中观察方向为 `b̂` 的像素，给定太阳方向 `â`：
1. 散射角为 `ω = arccos(â · b̂)`
2. 任何具有相同散射角的标准事件 `(â₀, b̂₀)` 都可能有贡献
3. 将 `(â₀, b̂₀)` 映射到 `(â, b̂)` 的旋转 `U` 是唯一确定的 (Gislén eq. 19)：

```
U = (â·â₀ᵀ + b̂·b̂₀ᵀ - cosω(â·b̂₀ᵀ + b̂·â₀ᵀ) + (â×b̂)(â₀×b̂₀)ᵀ) / sin²ω
```

4. 这个 `U` 就是使该标准事件产生观测到的散射所需的晶体朝向。评估 `Q(U)`——晶体
   朝向分布——得到权重。

像素值是所有匹配标准事件的加权和：

```
pixel_value = Σᵢ wᵢ × Q(Uᵢ)
```

## 算法设计

### 阶段一：预计算（每种晶体形状做一次）

在 S² 上密集采样入射方向 `â₀`（如 ~100K–500K 个方向）。对每个 `â₀`，在标准朝向下
光线追踪晶体，记录所有光路：

```
PrecomputedEvent {
    â₀: Vec3           // 标准帧下的入射方向
    b̂₀: Vec3           // 标准帧下的散射方向
    ω: float           // 散射角 = arccos(â₀ · b̂₀)
    weight: float      // Fresnel × 投影面积
    raypath_id: int    // 光路编号（用于 filter 支持）
}
```

按 `ω` 排序所有事件存入单一数组。这就是**预计算事件仓库**。

### 阶段二：渲染（逐像素，确定性）

对每个像素：

1. **确定 ω 范围**：像素对应有限立体角，映射到一个范围 `[ω_min, ω_max]`。
   可通过亚像素采样（在像素角点处计算 `arccos(â · b̂)` 取极值）得到，
   也可通过投影的 Jacobian 解析推导。

2. **范围查询**：在排序后的事件仓库中二分查找 `ω ∈ [ω_min, ω_max]` 的所有事件。
   查找 O(log N)，遍历 O(K)（K 为匹配数）。

3. **加权累积**：对每个匹配事件 `(â₀ᵢ, b̂₀ᵢ, wᵢ)`：
   - 通过 eq. 19 计算旋转 `Uᵢ`，使 `(â₀ᵢ, b̂₀ᵢ) → (â, b̂)`
   - 评估 `Q(Uᵢ)`——晶体朝向概率
   - 累加：`pixel_value += wᵢ × Q(Uᵢ)`

4. **归一化**：除以预计算入射方向数（或对应的立体角度量）得到物理正确的强度。

### 数据结构

```
// 按 omega 排序——唯一需要的数据结构
std::vector<PrecomputedEvent> events;  // 按 ω 排序

// 逐像素查询
auto [lo, hi] = equal_range(events, omega_min, omega_max);
for (auto it = lo; it != hi; ++it) {
    Mat3 U = compute_rotation(it->a0, it->b0, sun_dir, pixel_dir);
    float q = orientation_distribution(U);
    pixel_value += it->weight * q;
}
```

## 核心特性

### 自调节亮度

算法天然产生正确的亮度分布：

- **明亮的焦散**（如 22° 冰晕边缘）：在最小偏向角附近，`dω/dâ₀ → 0`，大量标准事件
  聚集在相近的 `ω` 值。范围查询返回大量匹配 → 像素明亮。这是物理正确的——焦散亮度
  正是来自同样的聚集效应。

- **暗区域**：很少有标准事件散射到这些角度 → 匹配少 → 像素暗淡。与 MC 行为一致
  （很少有光线命中这些像素），但没有噪声。

- **定向晶体**：`Q(U)` 起到筛选作用——只有与晶体分布一致的朝向才有显著贡献。幻日、
  切弧等朝向相关特征自然涌现。

### 预计算复用

事件仓库仅依赖晶体几何形状和折射率。以下任何变化只需重新运行阶段二（廉价），
而不需要阶段一：

- 太阳高度角 / 方位角
- 晶体朝向分布 `Q(U)`（倾斜角、滚转分布）
- 图像分辨率、投影方式、视场角
- 波长（如果按波长预计算了独立仓库）

这对 GUI 交互式参数调节尤其有价值。

### 无需插值

与需要等值线提取或 S² 上插值的方法不同，范围查询方法直接操作离散的预计算事件。
像素的有限面积天然定义了查询范围，离散求和逼近连续积分。

如果范围查询返回零匹配，意味着：
1. 预计算太稀疏（需增加采样密度），**或者**
2. 该像素确实强度极低（该散射角处的微分截面很小）

实际中情况 2 占主导——无匹配的像素在 MC 下同样是暗的。

## 成本分析

### 预计算

| 参数 | 典型值 |
|------|--------|
| 入射方向数 | 100K–500K |
| 每个方向的光路数 | ~10 |
| 总事件数 | ~1M–5M |
| 单次光线追踪成本 | ~200 ops |
| 总计 | ~2×10⁸–10⁹ ops（相当于一次 1–5M 光线的 MC 运行） |

这是每种晶体形状的一次性成本。

### 渲染

| 参数 | 典型值 |
|------|--------|
| 图像像素数 | 10⁶ (1000×1000) |
| 每像素匹配事件数 | ~100–1000（随 ω 变化） |
| 每次匹配成本 | ~100 ops（旋转计算 + Q 评估） |
| 每次渲染总计 | ~10⁸–10⁹ ops |

与 MC 对比：
- 10M 光线 × ~200 ops = 2×10⁹ ops → 有噪声图像
- 100M 光线 × ~200 ops = 2×10¹⁰ ops → 较干净图像
- 逆向渲染：~10⁹ ops → 无噪声图像

### 参数扫描

探索 N 组参数组合（如不同倾斜角或太阳高度）时：
- MC: N × (完整 MC 运行) = N × 2×10⁹
- 逆向: 1 × 预计算 + N × 渲染 = 10⁹ + N × 10⁹

当 N > 2 时，逆向渲染已经更高效。

## 扩展：发散光冰晕

同一个预计算事件仓库也可用于发散光（近光源）冰晕模拟，如街灯冰晕。核心区别在于
散射角如何随像素变化。

### 平行光 vs. 发散光

| | 平行光（太阳） | 发散光（街灯） |
|---|---|---|
| 入射方向 | 所有晶体相同的固定 `â` | 随晶体空间位置变化 |
| 每像素 ω | 几乎恒定（一次范围查询） | 沿视线射线连续变化 |
| 空间积分 | 不需要（所有晶体等价） | 必须沿视线射线积分 |

### Ray Marching 方法

对于发散光冰晕，每个像素对应一条穿过 3D 空间的视线射线。位于射线上位置 **x** 处的
晶体与位于位置 **y** 处的晶体，看到光源的角度不同。因此散射角 `ω` 沿视线射线连续变化。

渲染过程变为：

1. 对每个像素，从观察者向晶体云中投射视线射线。
2. 定义包围晶体云的 near-far 范围 `[t_near, t_far]`。
3. 沿射线逐步步进。在每个步进位置 `t`：
   - 计算局部入射方向 `â(t)`（从光源到位置 `t`）
   - 计算局部观察方向 `b̂(t)`（从位置 `t` 到观察者）
   - 计算局部散射角 `ω(t) = arccos(â(t) · b̂(t))`
   - 在预计算事件仓库中对 `ω(t) ± δω` 做范围查询
   - 对每个匹配事件，计算 `U`，评估 `Q(U)`，乘以 `1/|x(t)|²`（光源的
     平方反比衰减），累加
4. 汇总所有步进的贡献 → 像素值。

沿射线的相邻步进具有相似的 `ω` 值，因此查询范围大量重叠。可以利用这一点：
对射线的完整 `ω` 范围一次性排序匹配结果，然后随 `t` 推进滑动窗口。

### 工程简化

- **Near-far 裁剪**：仅在晶体云范围内步进。距观察者很远的晶体贡献可忽略
  （来自光源和观察者的双重平方反比衰减）。
- **自适应步长**：在 `ω(t)` 变化缓慢处使用粗步长，在观察者附近（`ω` 随位置
  快速变化处）使用细步长。
- **ω 范围批处理**：不做逐步查询，而是将整条射线的 `ω` 范围划分为若干子范围，
  每个子范围查询一次，然后将匹配结果分配到对应的射线步进中。

### 与 Gislén 雪茄方法的比较

Gislén 原始的发散光算法使用 Minnaert 雪茄几何——对每个标准事件，在雪茄曲面上生成
点并评估 `Q(U)`。这在数学上很优雅，但需要理解和实现雪茄参数化、双极坐标系以及
相关的事件复用逻辑。

Ray marching 方法复用与平行光场景相同的预计算事件仓库和相同的范围查询逻辑。
唯一的新增部分是沿视线射线的空间循环。这在工程上更简单、更容易理解，代价是计算量
稍高（每个像素多次范围查询而非一次）。

### 成本估算

对于每条射线 S 个步进的发散光渲染：

| 参数 | 典型值 |
|------|--------|
| 图像像素数 | 10⁶ |
| 射线步进数 | ~50–200 |
| 每步匹配数 | ~100–500 |
| 每次匹配成本 | ~100 ops |
| 每次渲染总计 | ~10¹⁰–10¹¹ ops |

这比平行光场景贵 10–100 倍，但仍然是确定性和无噪声的。作为对比，Gislén 原始的
基于 MC 的发散光模拟同样比平行光 MC 需要更多计算量，原因是额外的空间维度。

## 注意事项与局限

### 波长依赖

折射率随波长变化，影响 `b̂₀` 和 `ω`。可选方案：
- 按波长（或 RGB 通道）分别预计算：成本 ×3
- 在几个参考波长间插值

### 多次散射

预计算事件仅代表单次散射。多次散射需要链式组合事件或混合方案
（首次散射用逆向渲染，后续散射用 MC）。

### 数值稳定性

旋转公式 (eq. 19) 在 `ω → 0`（前向散射）或 `ω → π`（后向散射）时退化，
因为 `sin²ω → 0`。这些区域需要特殊处理或截断。

### 预计算密度

S² 上的采样密度必须足以解析散射函数的快速变化，特别是全内反射边界和光路切换
附近。这些区域可能需要自适应加密。

## 建议的验证计划

1. **单光路原型**：仅对一条光路实现（如 22° 冰晕的棱面折射路径，面 3→1）。
   与相同太阳高度下的 MC 输出对比。

2. **随机朝向测试**：设 `Q(U) = const`，验证径向亮度轮廓与 MC 一致
   （应重现 22° 最小偏向角峰值）。

3. **定向晶体测试**：使用板状晶体分布，验证幻日出现在正确位置且相对亮度正确。

4. **性能基准测试**：测量预计算时间和单次渲染时间。与等质量（PSNR）MC 对比。
